## Overview

Connect Hello to your favorite third-party services to automate workflows, process payments, and track events in real-time. You can integrate payment gateways like Stripe, scheduling tools like Calendly, and even LLM providers for intelligent copilots. Webhooks enable instant notifications for key events such as new bookings or completed payments.

<Callout kind="tip">
Start with webhooks for real-time data sync. Test integrations in sandbox mode before going live.
</Callout>

## Webhook Setup

Set up webhooks to receive real-time events from Hello, such as `booking.created` or `payment.succeeded`. Provide a secure endpoint on your server.

<Steps>
  <Step title="Create Webhook" icon="plus">
    In your Hello dashboard, navigate to Settings > Webhooks. Click "New Webhook".
  </Step>
  <Step title="Configure Endpoint" icon="link">
    Enter your endpoint URL, e.g., `https://your-webhook-url.com/hello-events`. Select events to subscribe to.
  </Step>
  <Step title="Verify and Test" icon="check-circle">
    Hello sends a verification challenge. Respond with HTTP 200. Test with a sample event.
  </Step>
</Steps>

### Webhook Payload Example

Expect JSON payloads with event data.

<Request tabs="JavaScript,cURL" show-lines="true">
  ```javascript
  // Verify signature with Hello's secret
  const signature = req.headers['x-hello-signature'];
  const isValid = verifySignature(payload, signature, WEBHOOK_SECRET);
  if (!isValid) return res.status(401).send('Unauthorized');
  ```
  ```bash
  curl -X POST https://api.example.com/v1/webhooks \
    -H "Content-Type: application/json" \
    -H "X-Hello-Signature: v1=abc123..." \
    -d '{"event": "booking.created", "data": {"id": "evt_123"}}'
  ```
</Request>

<Response tabs="200">
  ```json
  {
    "event": "booking.created",
    "id": "evt_456",
    "timestamp": 1699123456,
    "data": {
      "patientId": "pat_789",
      "appointmentTime": "2024-01-15T10:00:00Z"
    }
  }
  ```
</Response>

<ParamField path="event" param-type="string" required="true">
  Event type, e.g., `booking.created`.
</ParamField>

<ParamField header="X-Hello-Signature" param-type="string" required="true">
  HMAC signature for verification. Use `webhook_secret` from dashboard.
</ParamField>

## Custom Dashboard Configurations

Tailor dashboards for verticals like medical booking.

<Expandable title="Medical Booking Dashboard" default-open="true">
  Customize widgets for patient intake and no-show tracking.

  ```json
  {
    "widgets": [
      {

        "filter": "status=pending"

      {
        "type": "metrics",
  ```

## LLM Copilot Connections

Integrate LLMs for automated support responses.

<CodeGroup tabs="OpenAI,Anthropic">
  ```javascript
  // OpenAI integration
  const openai = new OpenAI({ apiKey: 'YOUR_OPENAI_KEY' });
  const completion = await openai.chat.completions.create({
    model: 'gpt-4o-mini',
    messages: [{ role: 'user', content: 'Suggest follow-up for patient no-show.' }]
  });
  ```
  ```javascript
  // Anthropic Claude
  const response = await fetch('https://api.anthropic.com/v1/messages', {
    method: 'POST',
    headers: {
      'x-api-key': 'YOUR_ANTHROPIC_KEY',
      'anthropic-version': '2023-06-01',
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      model: 'claude-3-5-sonnet-20240620',
      max_tokens: 1024,
      messages: [{ role: 'user', content: 'Generate sales email.' }]
    })
  });
  ```
</CodeGroup>

## Best Practices

- Always verify webhook signatures to prevent replay attacks.
- Use sandbox environments for testing integrations.
- Monitor integration logs in the Hello dashboard for errors.
